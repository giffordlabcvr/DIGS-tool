The DIGS tool

The database-integrated genome-screening (DIGS) tool is an open source PERL program for implementing systematic in silico genome screening.

What is in silico genome-screening?

Molecular sequence data are highly information rich, and are now being generated much faster than they can be analysed. Consequently, published genome sequences contain many features (e.g. genes, pseudogenes, transposons) that have not been yet been formally described, or are unreliably annotated.

Sequence similarity search tools, such as those implemented in the Basic Local Alignment Search Tool (BLAST) can be used to systematically screen whole genome data and recover sets of related sequences without relying on previously generated genome annotations.

This approach can form the basis for comparative studies focussing on specific genome features such as genes, mobile genetic elements, or conserved protein domains. In addition, similarity searches can in themselves provide useful insights into the distribution of conserved sequence features, for example by revealing differences in gene copy number between species.

What is database-integrated genome-screening (DIGS)

In database-integrated genome-screening (DIGS), the output of sequence similarity search-based screens is captured in a relational database This allows for the interrogation and manipulation of output data using structured query language (SQL).

Requirements

Similarity searches: the DIGS tool uses the Basic Local Alignment Search Tool (BLAST) to perform sequence similarity searches.

Relational Database: the DIGS tool uses the MySQL relational database management system (RDBMS) to coordinate in silico screens and to capture their output.

Instructions for installing and running the DIGS tool can be found on the DIGS tool wiki
