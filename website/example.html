How to run the DIGS tool

This tutorial assumes that you have installed the DIGS tool and comprises the following steps:

  1. Set up target library
  2. Set up reference library
  3. Set up probe library
  4. Set up a control file
  5. Run a screen
  6. Inspecting results
  7. Update reference sequences and reassign
  8. Consolidate


Set up target library

The DIGS tool is designed to perform systematic, simialrity-search based screening of local DNA sequence 'databases' - in other words, collections of FASTA formatted DNA sequences. This is the form that most genome and transcriptome assemblies are distributed.

The DIGS tool uses the BLAST program suite to perform similarity searches. BLAST can be used to search any file containing FASTA-formatted sequences for similar sequences.


Set up reference library

DIGS requires a library of FASTA-formatted reference sequences for the gene or genetic element under investigation. Probes for screening are selected from the reference library.

The DIGS tool uses a simple rule to capture data from the headers of FASTA-formatted reference (and probe) sequences; headers should be structured so as to define two hierarchical name elements; ‘name’ and ‘gene_name’, separated by an underscore. In the example shown above these are a virus name and a gene name. Other two-level hierarchical naming schemes (e.g. species & gene name, gene-subdomain name) can also be used, providing the same scheme is used consistently throughout the project. Reference sequences should be stored in a file, the path to which will be specified in the DIGS control file.


Set up probe library

A subset of reference sequences should be selected as probes. The entire reference library can be used - in which case there is no need to create a separate file – but it is often sufficient to use only a subset of sequences from the reference sequence library, in which case, a separate file containing this subset should be created. The path to this file is specified in the DIGS control file.


Set up control file

The DIGS control file contains parameters and paths for DIGS. Control files are structured as NEXUS style blocks delineated by BEGIN and ENDBLOCK tokens. An annotated example of a DIGS control file is shown below. Note that not all parameters will need to be defined for every screen.


Run a screen

Run the DIGS tool as follows. 

   ./digs_tool.pl -m=2 -i input.ctl
